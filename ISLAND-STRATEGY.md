### 1.3 食材手（核心分析單位）

- **定義：** 玩家手上**累積擁有的食材專業寶可夢總數**（與「樹果手」「技能手」對立的精靈池）。
- **量化方式：** 一個整數 N，代表玩家目前已抓到 / 已養成可用的食材手寶可夢有幾隻。隨著刷各島睡姿、抓到新精靈、舊精靈升技/進化到堪用狀態，N 會持續累積。
- **關鍵門檻（Wallace 提供）：**

  | 食材手累積數 | 狀態 | 備註 |
  |--------------|------|------|
  | < 9 | **未起跑** | 還湊不出穩定料理隊，談不上「畢業」；先把基本盤撐起來。 |
  | 9–11 | **起步區** | 能做基礎料理，靠保底配方撐鍋子和等級不夠的前期。 |
  | **12** | **畢業咖啡料理** | 可穩定產出咖啡系列當前最高料理所需食材。 |
  | **13** | **畢業酪梨料理** | 可穩定產出酪梨系列當前最高料理所需食材。 |
  | 12 / 13 | **共同價值** | 前期鍋子容量 / 隊伍等級不足時，這兩個畢業點能保底「至少有一道高 CP 值料理可煮」。 |

- **食材手的次要屬性（每隻精靈內部變數）：**
  - 食材槽位（1 / 2 / 3 個食材選擇）——影響可用食材種類
  - 主食材類型（每隻固定 1–3 種食材）
  - 等級 / 技能等級——影響單位時間產出量
- **「計入 N」的養成門檻（Wallace 提供）：**
  - **30 級 + AA 性格與副技能** = 入帳，可以算進 N
  - **60 級 + AAA** = 該食材手的成熟版本，但**不強求立即推到 60**
  - 換句話說：30 AA 是「能用」的起點，60 AAA 是「滿配」的終點，分析時以 30 AA 為計入下限。
- **與「畢業」的關係：** 見 §1.4。

### 1.4 畢業（Wallace 的官方定義）

> **畢業 = 三系料理（咖哩濃湯 / 沙拉 / 點心飲料）的當前最高料理，都能由同一個食材手組合穩定產出所需食材的狀態。**

- 「三系畢業」是個**團隊配置等級**，不是個體寶可夢的養成等級。
- **咖啡畢業（N = 12）** 的核心 3 道菜（Recipe Excel 已確認）：
  - 覺醒力量濃湯（咖哩）：萌綠大豆×28 / 好眠番茄×25 / 品鮮蘑菇×23 / 醒腦咖啡豆×16
  - 不服輸咖啡沙拉（沙拉）：醒腦咖啡豆×28 / 豆製肉×28 / 純粹油×22 / 窩心洋芋×22
  - 土王閃電泡芙（點心）：放鬆可可×30 / 哞哞鮮奶×26 / 醒腦咖啡豆×24 / 甜甜蜜×22
  - 三道合計需 **10 種不重複食材**（每種至少配一隻食材手）→ 加上中高階補位精靈 = N=12。
- **酪梨畢業（N = 13）** 的核心 3 道菜：
  - 茂盛焗烤酪梨（咖哩）：嫩亮酪梨×22 / 窩心洋芋×20 / 哞哞鮮奶×41 / 純粹油×32
  - 重踏酪梨醬脆片（沙拉）：嫩亮酪梨×28 / 萌綠玉米×25 / 火辣香草×30 / 萌綠大豆×22
  - 採蜜巧克力格子鬆餅（點心）：甜甜蜜×38 / 萌綠玉米×28 / 純粹油×28 / 放鬆可可×21
  - 三道合計需 **9 種不重複食材** → 加上中高階補位精靈 = N=13。
- 12 / 13 具體**精靈名單**：Wallace 將下次補充。
- 評估 A/B 路線優劣的最終標尺：**哪條路線更快、更穩定地把玩家的食材手累積數從現有 N 推進到下一個關鍵門檻？**

### 1.5 駐守模式

| 模式 | 操作意義 |
|------|----------|
| **長駐（Full Coverage）** | 在該島依序刷完淺淺、安然、深深三種睡姿的所有獨佔寶可夢（最完整食材池）。 |
| **深深駐守（Deep-Only）** | 只挑該島熟睡睡姿獨佔的寶可夢來補位（速度快，種類窄）。 |
| **輪換（Rotation）** | 已長駐 / 深深駐守過的島之間切換，按當週需要的食材類型選島睡眠。 |

## 2. 核心問題與決策框架

### 2.1 核心問題

**在追求「三系料理畢業」的過程中，玩家應該選哪條換島路線？**

具體分歧點：拿到第五、六、七島後，要把每個島都長駐到底（B 路線），還是只長駐五島、六七島用深深快速補位再回頭輪換（A 路線）？

### 2.2 兩條路線的「形狀」

**A 路線 — 混合換島（Mixed Rotation）**

```
五島：長駐 → 六島：深深駐守 → 七島：深深駐守 → 五/六 輪換
```
重點：五島打基底，六七島只挑熟睡精靈快速擴大食材**種類**，後期靠輪換配料理需求。

**B 路線 — 完全長駐（Full Residence）**

```
五島：長駐 → 六島：長駐 → 七島：長駐
```
重點：每個島都吃完整三睡姿，深度優先。

### 2.3 兩條路線的真正差別在哪

不是「速度 vs 穩定」這種抽象維度，而是這幾個可量化的問題：

1. **從現在的食材手累積數 N，到下一個關鍵門檻（9 / 12 / 13），A 路線比 B 路線快幾天？**
2. **A 路線只刷六/七島熟睡精靈，是不是漏拿了 12（咖啡）或 13（酪梨）畢業所需的關鍵食材手？** 如果關鍵精靈卡在六島淺淺/安然才出現，A 路線會直接卡關。
3. **B 路線在六/七島完整刷三睡姿，這段時間 A 路線玩家已經在輪換產出食材了，這個輪換期的料理產能差，能不能彌補 B 路線的精靈總數優勢？**

> 這三個問題就是 §4 數據分析要回答的東西。

## 3. 預期結論的形狀（先寫出來，數據驗證後再調整）

> 假設性結論，**Wallace 提供 Excel 後需要驗證 / 推翻**：

按玩家**當前**食材手累積數 N 分區：

- **N < 9（未起跑）**：A/B 之爭暫時無意義；優先湊滿 9 隻起跑線。哪條路線能最快把 N 推到 9，哪條就贏。
- **N = 9–11（起步區）**：拼速度上 12 / 13 兩個畢業點。**A 路線**靠六/七島熟睡精靈快速擴展食材手總數，理論上推進到 12/13 較快——但前提是缺的精靈在熟睡睡姿能拿到。
- **N = 12（咖啡畢業後）→ 13（酪梨畢業）**：分岔點。若卡在 12 的玩家發現酪梨需要的關鍵精靈必須長駐六島淺淺/安然才出，**B 路線**反而是唯一解。
- **N ≥ 13（雙畢業後）→ 完整三系畢業**：兩條路線的差距開始模糊；看追求的是「再多幾種彈性食材」（A 路線輪換取勝）還是「把單一精靈練到滿級」（B 路線長駐有時間優勢）。

**A/B 分岔點**：預期落在 **N = 12 到 13 之間** 的某個位置，但實際斷點要靠 Excel 驗證。

⚠️ 上面所有「A 路線快」「B 路線唯一解」都是**待驗證假設**，不是結論。
