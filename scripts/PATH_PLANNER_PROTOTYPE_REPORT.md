## Theory

```
σ_focused(O_t, scope) = Σ_type max{ score_final(r) : r cookable in O_t, r.type=type, r.ings ⊆ scope }
R(π) = Σ_t σ_focused(O_t, scope)  [optionally / n^α for patience]
Optimal path π* = argmax R(π), subject to: required ⊆ O_n
```

- **Scope** = the ings the user is willing to grind. Default = `final_ings` (just the goal-recipe ings).
- **Extras** = optional ings the user authorizes — they extend `scope` and let more recipes count toward σ.
- **σ_focused vs σ_free**: σ_focused only counts recipes whose ings ⊆ scope. **σ_free was the bug** — it counted any cookable recipe, leading the algorithm to detour through high-tier recipes outside the user's goal (e.g., Coffee 3 detoured through avocado +78%).
- **Local** = K=1 greedy (per-step σ-gain max, tie-break by appears-in count)
- **Global** = K=20 beam search ranking partial paths by `R(prefix)` + optional 2-opt polish
- **Patience α**: divide R by `n^α` — penalize longer paths.

## What's defensible vs still arbitrary

**Defensible (data-driven, no hand-tuned weights):**
- σ_focused — pure max-per-type of in-scope cookable recipes, using game's own `score_final`
- R(π) — sum of σ over the path
- Greedy / Beam / 2-opt — standard search algorithms
- Scope frontier — exhaustive enumeration of extra-subset choices

**Knobs the user controls (not arbitrary because user-chosen):**
- Final goals (3 dishes)
- Scope extras (which extra ings to grind for intermediate payoff)
- Algorithm: Local (greedy) or Global (beam K=20)
- Patience α ∈ [0, 1]: 0 = pure AUC, 1 = AUC per step

**Removed from previous design:**
- `worstBoost`, `wasteGlobal`, `coverageBonus`, `wasteLocal` weights — all arbitrary, gone.
- "中繼料理 pre-selection" — gone. Intermediates emerge from path as σ jumps.
- "Adjacency filter" — gone. Scope is explicitly chosen, not implicitly filtered.

## How to read the path output

Each scenario prints:
- **Sequence**: the ordered ing path (1 → 2 → 3 → …)
- **σ trajectory**: σ value after each step — the player's "standing" curve
- **In-scope milestones**: recipes that became cookable at each step, restricted to scope. These are the emergent intermediates.
- **Out-of-scope side-effects**: recipes that also become cookable but aren't counted in σ (e.g., your start-owned ings happen to enable some other recipe). Listed only as count; not noise.

## To use the prototype yourself

```
python3 scripts/path_planner_prototype.py
```

Edit `main()` to add scenarios. Each `report_scenario(...)` call takes:
- goal recipe names (list of 3)
- owned ings at start (optional)
- `alpha`: patience exponent
- `beam`: beam size for Global
- `extra_scope_ings`: optional manual scope extension

Use `scope_frontier(...)` to brute-force the Pareto frontier of extras for any goal.
